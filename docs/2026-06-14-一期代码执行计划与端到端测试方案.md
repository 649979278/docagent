# WorkAgent 一期代码执行计划与端到端测试方案

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 WorkAgent 一期在现有代码基础上完成可稳定运行的离线公文 Agent 最小闭环，优先打通 Runtime、Worker、Plan、RAG、Desktop UI 和测试链路。

**Architecture:** 继续沿用现有 Electron + Monorepo + AgentRuntime 分层，不推翻已有结构。优先修复真实运行路径中的断层，确保 Worker 与主进程运行行为一致，知识库索引与检索可持久化，Plan 模式从 UI 到 Runtime 到存储形成完整闭环。

**Tech Stack:** Electron、React 19、TypeScript、pnpm workspace、sql.js、LanceDB、Ollama、本地 OpenAI-compatible Provider、Vitest、Node 24、electron-builder

---

## 一、范围定义

本计划只覆盖“一期可交付闭环”，不直接展开二期增强中的高级能力。

一期完成标准：
- Mac 开发环境可稳定运行开发、单测、集成测试、桌面冒烟测试。
- Worker 模式与 direct 模式功能一致。
- 本地知识库可导入、索引、检索、重启后保留。
- Plan 模式可从 UI 打开、生成计划、审批、进入执行、导出文档。
- Electron 桌面壳可在 Mac 本机运行验证。
- 可从 Mac 产出 Windows `nsis` 安装包，至少完成打包链路和静态验收。

非一期目标：
- 暂不做多租户。
- 暂不做远程协作。
- 暂不做复杂 reranker / hybrid retrieval / 评测平台。
- 暂不做类似 Codex App 的完整 IDE 工作区能力。
