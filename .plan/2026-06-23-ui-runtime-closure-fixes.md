# 2026-06-23 UI 与运行时闭环修复

## 目标
- 修复新会话误显示计划审批的问题。
- 修复删除/归档后的会话跳转与归档恢复。
- 将环境信息、输入区菜单改为真正悬浮层，避免被裁剪。
- 校正标题栏、输入区和项目菜单的桌面端对齐。
- 确保计划模式、权限策略、模型切换、思考强度都真正作用于运行时。

## 实施步骤
1. 增加通用 `FloatingLayer`，统一解决底部菜单、环境弹窗、项目菜单的定位和遮挡。
2. 在 `run-store` 增加会话级运行态重置能力，创建/删除/恢复会话时统一清理计划残留状态。
3. 将计划审批从对话页移到右上角环境浮窗，避免新对话界面被计划 UI 污染。
4. 接通归档恢复后的会话重新选择，保证设置页恢复后立即回到项目上下文。
5. 让 `reasoning_level` 变更触发 runtime 重初始化，保证模型参数切换真实生效。
6. 调整标题栏、输入框和悬浮菜单的尺寸与对齐，再做桌面端实机验证。

## 验证点
- 新建会话后不再出现计划审批面板。
- Shift + Tab 可切换计划模式，计划审批只在环境浮窗显示。
- 删除/归档当前会话后，能跳到同项目下首个可用会话。
- 设置页可恢复归档会话，并立即回到对应项目上下文。
- 底部加号、权限、模型菜单和右上环境浮窗均不再被遮挡。
- 切换模型与思考强度后，下一轮运行使用新配置。
