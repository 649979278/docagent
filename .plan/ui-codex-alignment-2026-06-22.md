# Codex UI 对齐闭环记录

## 目标

按 Codex App 截图对齐 WorkAgent 桌面端交互：项目文件夹、环境浮窗、输入区权限和模型菜单、中文原生菜单、真实设置面板。

## 已落地

- 左侧项目区使用文件夹图标，新增项目菜单包含“新建空白项目”和“使用现有文件夹”，通过真实目录选择和 workspace IPC 创建项目并绑定当前会话。
- 初始页面插图改为文件夹语义。
- 右上环境信息去掉非操作按钮，加入当前目录、模型、上下文圆环、知识库数量、模式/计划阶段和摘要。
- 输入区加号改为 Add 菜单，目前提供文件导入和计划模式切换；底部移除“引用技能/计划模式”文字按钮。
- 权限菜单实现三档策略：每次询问、仅危险询问、完全访问，并持久化到 settings。
- 模型菜单读取本地模型列表并持久化当前模型，模型变更触发 runtime/worker 重初始化。
- Electron 原生菜单改为中文：文件、编辑、视图、帮助，并接入新对话/打开项目命令。
- 设置按钮打开真实设置面板，支持明暗/跟随系统、字体缩放、知识库导入、模型地址与模型切换。

## 验证

- `pnpm --filter @workagent/desktop build` 通过。
- `pnpm desktop:prepare-native` 通过，`electron_native: ok`。
- 本地服务已重启，Vite `http://localhost:5173/` 可访问，Electron Worker 模式初始化成功。
- 浏览器冒烟结构正常，唯一 console error 是 `favicon.ico` 404，不影响业务。
