# UI 标题栏与底部菜单清理记录

## 目标

按用户反馈修复六处 UI 问题：顶部中文菜单位置、无意义按钮、输入默认文案、底部弹窗遮挡、权限菜单文案与对齐、模型思考等级真实设置。

## 已落地

- Electron 隐藏系统菜单栏，renderer 新增顶部中文标题栏，`文件/编辑/视图/帮助` 与应用标题位于最顶行。
- 项目标题右侧只保留真实“添加项目”文件夹按钮，删除无意义图标。
- 中间对话标题行删除无意义三个点按钮。
- 输入框删除默认占位文案。
- 输入卡片取消 `overflow-hidden` 裁剪，底部 `+`、权限、模型菜单不再被输入框遮挡。
- `+` 菜单改为中文：文件和文件夹、计划模式。
- 权限按钮与浮窗重做：使用简洁状态图标、中文对齐文案、三档策略保持真实持久化。
- 模型菜单加入思考程度低/中/高，写入 `reasoning_level` 设置。
- 主进程将思考程度映射为真实模型请求参数：低 `temperature=0.35/maxTokens=2048`，中 `0.55/4096`，高 `0.7/8192`。

- `pnpm desktop:prepare-native` 通过，`electron_native: ok`。
- 本地 dev 服务已重启，Vite 可访问，Electron Worker 模式初始化成功。
- 扫描确认旧 UI 字符未残留：`Files and folders`、`盾`、`···`、`⌝`、`要求后续变更`、`引用技能` 等均已清理。
